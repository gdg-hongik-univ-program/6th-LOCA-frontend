# 🗺️ LOCA (로카) - 초개인화 로컬 라이프 플랫폼
> **"대중의 별점이 아닌, '내 스타일'로 동네를 디깅(Digging)하다."**  
> 🔗 **Live Demo**: [loca-hongik.vercel.app](https://loca-hongik.vercel.app/)

LOCA는 넘쳐나는 대중적 정보와 광고성 평점 피로감에서 벗어나, 사용자의 세밀한 취향과 방문 맥락을 학습하여 홍대 지역의 맞춤형 장소와 색다른 경험을 제안하는 반응형 웹 서비스입니다.

---

## 📱 서비스 핵심 기능 및 화면 구조

### 1. 메인 대시보드 (Home)
사용자의 현재 진입 목적에 따라 명확하게 두 가지 탐험 모드를 제안하고, 최근 트렌드를 요약하여 보여주는 관문 역할을 합니다.
* **큐레이션 헤더**: 접속 기기 환경에 맞춘 반응형 타이틀 및 알림 컴포넌트 배치.
* **네비게이션 카드**: 개인 맞춤형 공간인 **For You**와 탐색 공간인 **Explore**로 진입을 유도하는 직관적인 카드 UI 설계.
* **실시간 장소 피드 (PC 전용)**: 데스크톱 화면에서 하단에 '요즘 눈에 띄는 장소' 그리드 섹션을 노출하여 탐색 효율성 극대화.

### 📍 2. For You 모드 (초개인화 맞춤 추천)
사용자가 설정한 개인 취향 태그와 누적된 방문 데이터를 결합하여 작동하는 인텔리전트 추천 탭입니다.
* **추천 및 검색 로직**: 대중적인 평점이 아닌 사용자가 선호하는 분위기, 방문 목적(ex: `#혼술하기좋은`, `#새벽영업`)과의 일치도를 정밀하게 연산하여 장소 큐레이션 및 검색 결과를 매칭합니다.
* **컨텍스트 필터**: 사용자 맞춤형 장소 추천 결과를 스와이프 가능한 칩셋 인터페이스와 리스트 형태로 시각화.

### 🌟 3. Explore 모드 (지도 연동 인터랙티브 플랫폼)
지도를 중심으로 홍대 로컬 플레이스를 직접 검색하고 실시간 가로 스크롤 칩셋을 통해 공간을 탐색하는 하이브리드 탐험 탭입니다.
* **지도 API 연동**: 지도 API 연동을 통해 마커 핸들링 및 실시간 장소 위치 시각화 시스템을 구축했습니다.
* **고정형 상단 검색 바**: 어떤 스크롤 및 줌 상태에서도 빠르게 탐색할 수 있는 `absolute` 플로팅 검색 바 배치.
* **가로 롤 칩셋**: 카테고리 필터를 막힘없이 넘겨볼 수 있는 모바일 최적화 스크롤(`no-scrollbar`) 핏 구현.

### ✍️ 4. 경험 아카이빙 및 히스토리 관리 (방문 기록)
단순히 별점 수치만 기록하는 기존 지도 앱의 한계를 극복하기 위해 설계된 정성적 데이터 축적 시스템입니다.
* **맥락 기반 기록**: '누구와 방문했는지', '그날의 분위기는 어땠는지', '재방문 의사가 있는지' 등 개인의 라이프스타일 코드를 세부적으로 커스텀 저장.
* **데이터 선순환**: 저장된 기록은 다시 취향 분석 엔진의 가중치 데이터로 피드백되어 사용할수록 고도화되는 개인화 경험을 선사합니다.

## 🎨 UI/UX 및 반응형 아키텍처 특장점

* **하이브리드 반응형 레이아웃 (`AppShell`)**
  * 모바일 기기(768px 미만)에서는 하단 탭바(`BottomNav`)와 스크롤이 고정된 `100dvh` 풀스크린 웹앱 환경을 제공합니다.
  * 태블릿 및 데스크톱 환경에서는 왼쪽 영역에 고정식 사이드바 레이아웃 박스를 격리(Padding 확보)하여 화면 쏠림과 해상도 깨짐을 완벽하게 방어합니다.
* **터치 친화적 인터페이스**
  * 모바일 환경에서의 오버스크롤 바운스 차단(`overscroll-none`), 카테고리 탭 스크롤의 터치 제스처 최적화(`touch-auto`)가 적용되어 부드러운 네이티브 앱의 사용감을 전달합니다.

## 📚 기술 스택 & 데이터 아키텍처 (Tech Stack)

### 💻 Frontend & API
* **Framework**: Next.js (App Router)
* **Language**: TypeScript
* **Styling**: Tailwind CSS
* **API Configuration**: Next.js Route Handlers 기반의 독립적인 API 설정 및 비동기 데이터 통신 구조 설계
* **Deployment**: Vercel

### ⚙️ Database & Engine
* **Cloud Database**: **Supabase** (원격 클라우드 데이터 관리 인프라 구축 및 인증/실시간 데이터 연동)
* **Local Database**: **SQLite** (경량화된 관계형 데이터베이스 환경 기반의 데이터 관리 및 영속성 레이어 최적화)
* **DB Architecture**: 안정적인 관계 데이터 모델링 및 효율적인 스키마/인덱스 활용을 기반으로 한 **DB 설계** 적용
* **Logic Engine**: 데이터 파싱 및 사용자 맞춤형 **추천·검색 로직 자체 작성**
* **Map Engine**: **지도 API (Kakao Maps API / Naver Map API)**를 활용한 동적 맵 렌더링 및 인터랙션 구현
